## Representing the maze in code

To start with, we need a grid to work on. A grid is made up of cells, which we
can partition into rows and columns.

![Maze with all walls](/assets/images/2024-12-12-PGF-03/maze-all-walls.svg)

We can refer to columns using an X coordinate and rows using a Y coordinate
(0-indexed, of course!). For example, the highlighted cell in the below image is
the cell with X=4 and Y=6.

![Maze grid](/assets/images/2024-12-12-PGF-03/grid.svg)

We can represent this in code with `Cell` and `Grid` types.

```cs
public record Cell(int X, int Y);

public class Grid
{
    public Grid(int width, int height)
    {
        this.Width = width;
        this.Height = height;
        this.Cells =
            (from x in Enumerable.Range(0, this.Width)
             from y in Enumerable.Range(0, this.Height)
             select new Cell(x, y))
             .ToList();
    }

    public int Width { get; }

    public int Height { get; }

    public IEnumerable<Cell> Cells { get; }
}
```

We also need to keep track of which walls are present between cells. I've chosen
to represent this using the four cardinal directions on a map, but another
approach could be to store a wall as the pair of cells on either side of the
wall.

```cs
public enum Direction { North, East, South, West }

public record Wall(Cell Cell, Direction Direction);
```

I then have a `Maze` class which keeps track of which walls are actually present
